- **新发现的契约/约束**(候选写入 design-intent / PROJECT_CONTEXT):
  - **C1**: `registry_find_by_*` 借出内部指针,锁在返回前释放;caller 不可在并发触发 unregister 的情况下持指针。当前 IPC 线程单消费安全,**supervisor 阶段 2 上线时需重新评估**(若需长持元信息应复制)。
  - **C2**: paramiko 4.0 部署链路稳态 fallback = `tar + base64 over ssh stdin`(`PROJECT_CONTEXT` 条目 19 v6 已注 inline 用法,但本次再次验证 sftp 流式仍偶发 ENOENT,**应内化到 `scripts/deploy_target.py` 当 sftp 失败自动回退**)。
  - **C3**: 在 paramiko ssh exec 内启动 daemon 需 `setsid bash -c '...' < /dev/null > /dev/null 2>&1` 才能让 channel 干净退出 — 否则 `recv_exit_status` 挂起。属测试/部署侧隐性契约。
  - **C4**: dispatcher 内 ACK `write()` 是阻塞 socket;违反契约 14 的精神(IPC 线程虽不是 reactor 本身,但卡住会让新连接 dispatch 延迟)。**阶段 2 子程序监管上线前应改非阻塞写**。已在 `proto_dispatcher.c` 文件头 TODO 注释。
  - **C5**: register payload 业务层上限 = 4096(`parse_register_json` 内硬编码),**比协议层 `PROTO_MAX_PAYLOAD = 65536` 更紧**。建议固化为通用规则"业务层边界可以比协议层更紧,但不可超过协议层"。
