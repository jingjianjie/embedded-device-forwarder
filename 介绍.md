# embedded-device-router 项目说明

本项目对应仓库代号 **embedded-device-router**,核心交付物是运行在 ARM64 Linux 开发板(Orange Pi / Raspberry Pi,Debian)上的守护进程 **`ez_router`**。

> 旧版本曾以 C# / .NET 8 方案实现并被命名为 "Forwarder Daemon",该方案已废弃。当前实现为纯 C,源码位于 `routerd/`。本文是项目入门和术语规范的唯一权威。

---

## 一、术语约定

| 用语 | 定义 |
|---|---|
| **端侧 / Board** | "仪器 → 路由器 → 上位机" 拓扑中,**`ez_router` 所在的嵌入式开发板侧**。 |
| **路由器 / `ez_router`** | 本项目的核心守护进程,负责按 `config.json` 中的路由表,在多个本地端口之间原样转发数据。 |
| **上位机 / Host** | 运行在 PC 端的控制软件;与开发板通信并**主动发起 TCP 连接**的一方。绝不指代开发板。 |
| **下位机 / 设备(Device)** | 开发板自身,或开发板及其挂载的所有子设备整体,按上下文取义。 |
| **仪器 / 子设备** | 挂载在开发板上的物理设备(传感器、模组、相机、步进电机、光闸等)。 |
| **端口(Port)** | `config.json` 中 `ports[]` 的一项,可为 `tty` / `tcp_server` / `tcp_client` / `udp` / `usb` / `ipc`。 |
| **通道号 / Channel** | 帧协议中的逻辑编号,用于决定数据被路由到哪个挂载设备(选片)。 |
| **左侧 / 右侧** | 拓扑方向:**左**=靠近仪器,**右**=靠近上位机。 |
| **左侧链路** | `ez_router` 与仪器之间的链路。 |
| **右侧链路** | `ez_router` 与上位机之间的链路(通常 TCP)。 |
| **帧协议** | 右侧链路上传输数据的固定格式二进制协议。 |
| **选片(Chip Selecting)** | 借用术语,指根据帧头通道号决定数据流向哪个挂载设备。 |
| **插件(Plugin)** | `.so` 动态库,在路由命中时被调用以加工数据,按 `${插件名}.${函数名}` 注册。 |

---

## 二、拓扑

```
        [上位机 / PC]
              |
              | ← Host Link (TCP / IPC)
              |
   [ ez_router  on  Board ]
              |
   ---------------------------------
   |              |              |
仪器 A         仪器 B         仪器 C
 (UART)        (USB)          (TCP)
```

---

## 三、能力概述

- 多种端口类型并行接入: UART、USB CDC、TCP server/client、UDP、Unix Socket(IPC)
- 通过 `config.json` 声明端口、插件、路由规则,无需重新编译
- 支持运行时通过 IPC 命令通道(`/run/ez_router/ez_router.sock`)修改路由(SDK 见 `sdk/`)
- 路径上可挂载 `.so` 插件做数据加工/选片/过滤

## 四、源码组织

| 目录 | 职责 |
|---|---|
| `routerd/` | 守护进程 `ez_router` 的 C 源码 + cJSON。**主体**。 |
| `routerd/src/ez_router.c` | 进程入口、线程编排(reactor / dispatcher / ipc)。 |
| `routerd/src/reactor.c` | epoll 事件循环,负责所有端口 fd 的读取分发。 |
| `routerd/src/port_manager.c` | 端口抽象与生命周期(open / send / find)。 |
| `routerd/src/router_core.c` | 路由查表 → 目标端口写出。 |
| `routerd/src/plugin_loader.c` | `dlopen` 加载 `.so`,handler 自注册表。 |
| `routerd/src/config_store.c` | `config.json` 解析。 |
| `plugins/` | 路由插件源码,编译产物在 `out/plugin/`。 |
| `sdk/` | 上位机 C SDK,通过 IPC 与 `ez_router` 通信。 |
| `examples/` | 使用 SDK / IPC 通道的客户端示例。 |
| `scripts/manage_ez_router.sh` | systemd 服务安装/卸载/启停。 |

---

## 五、构建与运行

详见 `README.md`。简版:

```sh
cd routerd && make          # 产出 out/ez_router
cd ../plugins && make       # 产出 out/plugin/*.so
cd ../out && ./ez_router    # 配合本目录 config.json 运行
```
