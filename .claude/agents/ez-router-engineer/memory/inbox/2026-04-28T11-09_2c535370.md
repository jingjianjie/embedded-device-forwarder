- **关键方案偏离 — 主 agent 已确认接受**:
  - 原方案: `fcntl(fd, F_SETFL, O_NONBLOCK)` + EAGAIN 重试。
  - 实际落地: `send(fd, ..., MSG_DONTWAIT | MSG_NOSIGNAL)` per-call + EAGAIN 重试(8 次 × 1ms)。
  - 理由: `fcntl` 改 fd 属性会让**同 fd 的 recv 也变非阻塞** → IPC 线程 recv 路径 busy loop。`MSG_DONTWAIT` 只对当次系统调用生效,recv 阻塞语义保留。
  - 行为不变(都是非阻塞 + EAGAIN 退让),但实现路径正交于 fd 属性,这是契约 24 的实现说明级修正,主 agent 已落入 PROJECT_CONTEXT v8 §24。
